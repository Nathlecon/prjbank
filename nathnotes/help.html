Documentation API

Guide d'intégration

Tout ce qu'il faut pour connecter ton frontend à l'API NathNotes. Backend tourne sur le port 3001.

Table des matières
Authentification Gestion des notes Gestion des dossiers Exemples JSON

01

Authentification

NathNotes utilise des JWT tokens pour l'authentification. Le token est passé dans le header Authorization: Bearer .

POST /api/auth/register

Créer un nouveau compte utilisateur.

ChampTypeDescription
emailstringEmail unique
passwordstringMot de passe
namestringNom d'affichage
Requête
{ "email": "user@example.com", "password": "monmotdepasse", "name": "Jean Dupont" }
Réponse
{ "token": "eyJhbGciOiJIUzI1NiIs...", "user": { "id": 1, "email": "user@example.com", "name": "Jean Dupont" } }

POST /api/auth/login

Connexion avec email et mot de passe.

Requête
{ "email": "user@example.com", "password": "monmotdepasse" }

02

Gestion des notes

GET /api/notes

Récupérer toutes les notes de l'utilisateur (non supprimées).

Endpoint protégé — nécessite le token JWT

POST /api/notes

Créer une nouvelle note.

ChampTypeDescription
titlestringTitre de la note OPTIONNEL
contentstringContenu HTML OPTIONNEL
folderstringID du dossier OPTIONNEL (défaut: "all")
Réponse
{ "id": 1, "user_id": 1, "title": "Ma note", "content": "

Contenu...

", "folder": "work", "favorite": 0, "created_at": "2026-01-01 12:00:00", "updated_at": "2026-01-01 12:00:00" }

PUT /api/notes/:id

Modifier une note existante.

Requête
{ "title": "Nouveau titre", "content": "

Nouveau contenu

", "folder": "ideas", "favorite": 1 }

DELETE /api/notes/:id

Soft-delete (déplacer à la corbeille).

Soft delete — la note n'est pas définitivement supprimée, elle va dans la corbeille (deleted=1)

GET /api/notes/deleted

Récupérer les notes supprimées (corbeille).

POST /api/notes/:id/restore

Restaurer une note depuis la corbeille.

03

Gestion des dossiers

GET /api/folders

Récupérer tous les dossiers. Retourne les dossiers par défaut + les dossiers personnalisés de l'utilisateur.

Dossiers par défaut — All, Work, Personal, Ideas sont toujours présents (is_default: true)
Réponse
[ {"id":"all","name":"Tout","icon":"📁","is_default":true}, {"id":"work","name":"Travail","icon":"💼","is_default":true}, {"id":5,"name":"Mes projets","icon":"🚀","is_default":false} ]

POST /api/folders

Créer un nouveau dossier personnalisé.

ChampTypeDescription
namestringNom du dossier REQUIS
iconstringEmoji OPTIONNEL (défaut: 📁)
colorstringCouleur hex OPTIONNEL (défaut: #e8b84b)
Requête
{ "name": "Mes projets", "icon": "🚀" }

Guide d'import
de produits JSON

Tout ce qu'il faut savoir pour importer des produits en masse dans NathStore via un fichier ou du JSON collé directement dans le panel admin.

Sommaire
1. Format JSON attendu 2. Description des champs 3. Catégories disponibles 4. Exemples complets 5. Erreurs fréquentes 6. Conseils & astuces
01
Format JSON attendu

L'import attend un tableau JSON (array) contenant un ou plusieurs objets produits. Chaque objet représente un produit.

JSON — structure de base
[ { "nom": "Nom du produit", "emoji": "🎮", "categorie": "gaming", "prix": 29.99, "description": "Description courte", "badge": "NEW" }, { // Deuxième produit... } ]
Info : Tu peux importer un seul produit sans tableau — l'import accepte aussi un objet seul { ... } et le convertit automatiquement.
02
Description des champs

Voici tous les champs reconnus par l'API. Les champs requis doivent absolument être présents, les champs optionnels peuvent être omis.

ChampTypeRequis ?DescriptionExemple
nom string REQUIS Nom du produit affiché dans la boutique "Air Max 2026"
emoji string REQUIS Un seul emoji, affiché comme image produit "👟"
categorie string REQUIS Doit être exactement l'une des 4 valeurs valides "mode"
prix number REQUIS Prix en euros, décimal avec point (pas virgule) 89.99
description string OPTIONNEL Courte description affichée sous le nom. Peut être vide. "Sneakers confortables"
badge string | null OPTIONNEL Petit label rouge affiché sur la vignette. Mettre null pour aucun badge. "NEW" / "PROMO" / null
Attention : Si nom, emoji, categorie ou prix sont manquants, le produit est ignoré et compté dans les erreurs — les autres produits du même fichier sont quand même importés.
03
Catégories disponibles

Le champ categorie doit être exactement l'une de ces 4 valeurs (en minuscules) :

👟
Mode
"categorie": "mode"
💻
Tech
"categorie": "tech"
🎮
Gaming
"categorie": "gaming"
🍕
Food
"categorie": "food"
Erreur fréquente : Écrire "Mode" avec une majuscule ou "Mode 👟" avec l'emoji — l'API accepte uniquement les valeurs exactes en minuscules.
04
Exemples complets

Des exemples prêts à copier-coller directement dans le panel d'import.

Exemple minimal — 3 produits, champs requis uniquement :

JSON
[ { "nom": "Casquette NY", "emoji": "🧢", "categorie": "mode", "prix": 24.99 }, { "nom": "Souris Gaming", "emoji": "🖱️", "categorie": "gaming", "prix": 49.99 }, { "nom": "Box Ramen Premium", "emoji": "🍜", "categorie": "food", "prix": 22.50 } ]

Exemple complet — tous les champs :

JSON
[ { "nom": "Jordan 1 Low", "emoji": "👟", "categorie": "mode", "prix": 119.99, "description": "Colorway exclusif, édition limitée 2026", "badge": "LIMITED" }, { "nom": "Clavier Mécanique TKL", "emoji": "⌨️", "categorie": "tech", "prix": 129.99, "description": "Switches red, rétroéclairage RGB", "badge": null } ]
Astuce : Tu peux générer un fichier JSON avec ChatGPT ou Claude en demandant "génère moi 10 produits gaming au format NathStore JSON" — colle juste la structure des champs dans ton prompt.
05
Erreurs fréquentes
JSON invalide — SyntaxError
Virgule manquante entre deux objets, accolade non fermée, guillemets manquants autour d'une clé. Utilise un validateur JSON en ligne pour vérifier.
Produits ignorés — champs manquants
Un ou plusieurs produits ont été ignorés car nom, emoji, categorie ou prix est absent. Le message d'import affiche le nombre d'ignorés.
Catégorie invalide
La valeur de "categorie" n'est pas exactement "mode", "tech", "gaming" ou "food" en minuscules. Vérifie les majuscules et les espaces.
Prix avec virgule
En JSON, les décimaux s'écrivent avec un point : 29.99 et non 29,99. La virgule casse la syntaxe JSON.
Serveur inaccessible
Le backend Node.js n'est pas lancé. Ouvre un terminal et tape : cd projet-banque/banque && node index.js
06
Conseils & astuces

Valider son JSON avant d'importer — colle ton JSON sur jsonlint.com pour vérifier qu'il n'y a aucune erreur de syntaxe avant d'importer.

Importer par petits lots — si tu as 100 produits, importe-les en plusieurs fois de 20-30. En cas d'erreur, c'est plus facile à identifier.

Créer un fichier .json dédié — plutôt que de tout coller à chaque fois, crée un fichier produits.json dans ton projet et utilise l'onglet upload. Tu peux le modifier et le réimporter facilement.

Structure de fichier recommandée :

Arborescence
projet-banque/ ├── banque/ ├── banque-frontend/ ├── nathstore/ └── produits.json ← ton fichier de produits ici
Tip : Le badge peut être n'importe quelle courte chaîne de texte — "NEW", "PROMO", "TOP", "SOLD OUT", "LIMITED", "-20%"... Garde-le court (max 10 caractères) pour que ça rentre dans la vignette.
